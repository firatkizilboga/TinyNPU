# Third-Party Bare-Metal Baseline - onnx2c - 2026-05-19

Purpose: replace the handwritten Conv/TinyLM fp32 CPU baselines with a stronger
external path. The models are lowered to ONNX, converted to C by upstream
`onnx2c`, linked into the cv32e40p bare-metal harness, and run in RTL.

Library: `onnx2c` from https://github.com/kraiskil/onnx2c

Local tool:

```sh
external/third_party/onnx2c/build/onnx2c
```

Builder:

```sh
scripts/build_onnx2c_baremetal_baselines.py
```

Setup used on this machine:

```sh
git submodule update --init --recursive external/third_party/onnx2c
cmake -S external/third_party/onnx2c -B external/third_party/onnx2c/build -DCMAKE_BUILD_TYPE=Release -DBENCHMARK_ENABLE_TESTING=OFF
cmake --build external/third_party/onnx2c/build --target onnx2c -j 8
python3 -m venv .venv-onnx
.venv-onnx/bin/python -m pip install onnx numpy
```

## Notes

The MLP and Conv baselines are fully represented as ONNX models from input to
output.

The TinyLM baseline currently feeds the prompt embedding sequence as the ONNX
input and emits logits as the ONNX output. Tokenization, embedding lookup, and
argmax/token printing are in the small bare-metal wrapper. The transformer body
itself is generated by `onnx2c`: RMSNorm, RoPE, causal attention, softmax,
SwiGLU, residuals, final norm, and LM head matmul.

This is a stronger baseline than the handwritten fp32 C emitters because each
core model body is generated by an external ONNX compiler. It is still not a
full general-purpose runtime like TVM/TFLM: shapes are fixed at ONNX generation
time, and the TinyLM wrapper still owns prompt preparation and result reporting.
