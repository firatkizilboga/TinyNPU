# Model Execution Status - 2026-05-19

Purpose: consolidate which model executions are actually proven on RTL, which
ones are third-party CPU baselines, and which ones are still stale or broken.

Current interpretation:

- The external CPU baseline question is now covered for all three toy model
  families: MLP, Conv, and TinyLM.
- `onnx2c` is the strongest current baseline because it is an external compiler
  path and covers MLP/Conv/TinyLM, not only feed-forward MLP.
- TinyLM `onnx2c` still uses a small wrapper for tokenization, embedding lookup,
  and argmax reporting. The transformer body is generated by `onnx2c`.

Current interpretation:

- The NPU itself is not dead: GEMM, integration acceptance, QLlama, GPT2-like
  chains, and TinyStories prefill+decode all have passing RTL evidence.
- The stale MLP and Conv Runtime V2 artifacts are not current signoff points.
  They either fail verification or time out under the current RTL/runtime state.
- For thesis numbers, use the current passing transformer-shaped NPU results and
  clearly label older MLP/Conv speedups as historical unless those paths are
  rebuilt and revalidated.

## Clock-Adjusted Status

Clock assumptions from `runs/CLOCK_ADJUSTED_EXAMPLES_2026_05_18.md`:

- CPU-only routed point: `57.1 MHz`.
- CPU+NPU real-BRAM estimate: `39.17 MHz`.
- CPU+NPU needs more than `1.46x` cycle speedup to beat CPU-only wall time.

Current passing transformer examples:

| Model | Cycle speedup | Clock-adjusted status |
| --- | ---: | --- |
| QLlama decode `d32 h8 nh4 nkv2 f32 T8` hot | `1.30x` | still slower in wall time |
| QLlama decode `d48 h8 nh6 nkv3 f48 T8` hot | `1.77x` | faster in wall time |
| QLlama decode `d64 h16 nh4 nkv2 f64 T8` hot | `2.41x` | faster in wall time |
| GPT2 two-block `d16 h16 nh1 f16 T4` hot | `1.46x` | roughly break-even |
| GPT2 two-block `d24 h24 nh1 f24 T4` hot | `1.88x` | faster in wall time |

## What Remains

- Rebuild and revalidate the MLP NPU Runtime V2 path if MLP must be a current
  NPU result instead of a historical result.
- Rebuild and revalidate the Conv NPU Runtime V2 path with a controlled tick
  budget if Conv must be a current NPU result.
- Multi-head GPT2-like two-block lowering is not validated; the current passing
  GPT2-like chain is single-head only.
- `onnx2c` is fixed-shape generated C, not a dynamic runtime. It is good enough
  as an external bare-metal CPU baseline, but not a replacement for a full
  deployable ML runtime.
